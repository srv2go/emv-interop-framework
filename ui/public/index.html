Getting Started

üéØ Purpose

This framework helps you test Discover card specifications against various terminal types to ensure backward compatibility and interoperability.

Quick Start Guide:

  1. Run Tests: Go to the "Run Tests" tab and click on any test scenario to execute it
  2. View Results: Test results will show compatibility, warnings, and recommendations
  3. Custom Specs: Add your own card/terminal specifications in the Custom Specs tab
  4. Compatibility Matrix: Run the cross-vendor test to see full compatibility

Test Scenarios Explained:

Scenario What It Tests
D-PAS 1.0 on Legacy C6 Legacy card on old terminals (e.g., Verifone VX520)
D-PAS 2.1 on Modern C6 Current production cards with CDCVM support
D-PAS 3.0 C8‚ÜíC6 Fallback C8-ready cards falling back to C6 on legacy terminals

Adding Custom Applets:

To test your own Discover applet specification:

  1. Navigate to the custom specifications directory (shown in Custom Specs tab)
  2. Open the applets/template-discover-applet.json file
  3. Modify the values to match your applet specification
  4. Save with a new filename (e.g., my-dpas-applet.json)
  5. Click "Reload Custom Specifications" in the Custom Specs tab
  6. Your applet will now be available for testing
